# DFS와 BFS 핵심 개념

코딩 테스트에서 단골로 등장하는 그래프 탐색 알고리즘인 DFS와 BFS의 차이점과 기본 템플릿입니다.

## 1. DFS (Depth-First Search, 깊이 우선 탐색)
한 경로를 끝까지 탐색한 후, 다시 돌아와서 다른 경로를 탐색하는 방식입니다. **재귀 함수**나 **스택**을 활용하여 구현합니다.
경로의 특징을 저장해둬야 하는 문제(예: 각 정점에 적힌 숫자를 이어붙이는 경우)에 유리합니다.

```js
function dfs(graph, v, visited) {
  visited[v] = true;
  console.log(v);

  for (const node of graph[v]) {
    if (!visited[node]) {
      dfs(graph, node, visited);
    }
  }
}
```

## 2. BFS (Breadth-First Search, 너비 우선 탐색)
시작 정점으로부터 가까운 정점을 먼저 방문하고 멀리 떨어져 있는 정점을 나중에 방문하는 방식입니다. **큐(Queue)**를 활용하여 구현합니다.
가중치가 없는 그래프에서 **최단 경로**를 찾는 문제에 무조건적으로 사용됩니다.

```js
function bfs(graph, start, visited) {
  const queue = [start];
  visited[start] = true;

  while (queue.length > 0) {
    const v = queue.shift(); // JS 특성상 O(N)이므로 최적화 시 큐 직접 구현 필요
    console.log(v);

    for (const node of graph[v]) {
      if (!visited[node]) {
        queue.push(node);
        visited[node] = true;
      }
    }
  }
}
```

상황에 맞게 필요한 탐색 방법을 골라 쓰는 연습이 가장 중요합니다.
